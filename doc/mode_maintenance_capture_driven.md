# 方式1：抓包驱动（维护）

> 触发条件：任务类型为“维护”，且用户选择方式1，或表达“用抓包 / 读 latest.jsonl / 已完成 UI 操作 / 回溯最新链路”。

## 强制读取要求

进入本方式后，AI 必须先读取：

1. `doc/coding_style_guide.md`
2. `doc/maintenance_prompt_context.md`
3. 本文件 `doc/mode_maintenance_capture_driven.md`
4. 必要时读取 `capture/README.md` 排查抓包证书、代理或端口问题

## 适用场景

- 业务链路发生较大变化
- 一个变更影响多个接口或多个用例
- 需要按最新链路回溯现有方法与用例

## 执行步骤

1. **锁定受影响范围**：先确认需要维护的用例、接口方法和页面链路。
2. **判断抓包服务状态**：按现有抓包流程检查是否需要重启或启动。
3. **获取最新链路**：操作 UI 后读取 `latest.jsonl`，必要时重新生成索引与勾选草稿。
4. **对照现有实现**：查 `tools/page_api_index.sqlite3`，确认哪些接口方法可复用、哪些方法需要补丁式修改。
5. **定位维护点**：区分“只改断言”“只改参数”“方法与用例都要改”“需要新增一个替代方法”。
6. **按业务场景维护**：不要按抓包顺序机械拆分，按最终业务链路修补已有用例。
7. **执行 pytest**：只跑受影响用例或最小回归集，确认维护结果。

## 维护原则

- 优先修补已有方法，不先新建。
- 如果最新链路里出现替代接口，先判断是否可以沿用旧方法的调用方式再改。
- 当多个用例共用同一个方法时，优先确认公共影响面，再动公共方法。
- 维护场景下，允许在必要时新增接口方法，但新增只是维护手段，不是任务目标。
