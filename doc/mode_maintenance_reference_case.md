# 方式2：参考已有用例（维护）

> 触发条件：任务类型为“维护”，且用户选择方式2，或表达“参照 / 仿照 / 复制已有用例 / 参考现有用例修复”。

## 强制读取要求

进入本方式后，AI 必须先读取：

1. `doc/coding_style_guide.md`
2. `doc/maintenance_prompt_context.md`
3. 本文件 `doc/mode_maintenance_reference_case.md`
4. 用户指定的参考用例全文及其所属测试类头部

## 适用场景

- 已有用例结构稳定，只是参数、断言、fixture、前置依赖变化
- 需要借参考样本做局部修补
- 修复单个或少量用例，而不是重建整条链路

## 执行步骤

1. **锁定参考样本**：用户必须给出参考用例函数名、文件路径，或明确说明要对照的同类维护样本。
2. **读取参考上下文**：读参考用例全文与类头部，确认实例、fixture、参数化和断言风格。
3. **定位差异**：先判断变化只影响用例，还是同时影响接口方法。
4. **最小补丁维护**：尽量只改受影响的断言、参数、fixture 或方法调用，不扩大改动面。
5. **保持公共骨架**：如果同类用例共享结构，沿用原有骨架，不重新设计。
6. **执行 pytest**：优先跑目标用例或目标类的最小回归。

## 维护原则

- 参考样本只用于定位和对齐，不拿来生成新任务语义。
- 不因为维护而把结构改复杂。
- 若参考样本与最新链路冲突，以真实返回和最新链路为准。
